Rate

Rate API API Reference

An API that allows a user to get a parking rate for a time span.

API Endpoint
http://localhost:8080/
Schemes: http
Version: 1.0.0

metrics

Report of cumulative performance metrics for 'rate' API endpoints

metrics

GET /metrics

200 OK

Responds with detailed performance metrics per 'rate' API endpoint.

Response Content-Types: application/json

ping

Status-check endpoint to verify service is available

ping

GET /ping

200 OK

Responds with 'pong\n' when service available

Response Content-Types: text/plain;charset=iso-8859-1

rate

Parking rate for a time span

rate

Get a parking rate for a time span

GET /v1/rate
start: string (date-time)
in query

Start of the time span (inclusive). If 'start' is absent, in an improper format, exceeds 'end', or is on a different date than 'end', the rate will be unavailable. Regardless of the provided UTC offset, the day of the week will be relative to the time with a UTC offset of 0.

end: string (date-time)
in query

End of the time span (inclusive). If 'end' is absent, in an improper format, precedes 'start', or is on a different date than 'start', the rate will be unavailable. Regardless of the provided UTC offset, the day of the week will be relative to the time with a UTC offset of 0.

200 OK
Rate

Parking Rate

Response Content-Types: application/json
Response Example (200 OK) 
{
  "price": "integer (int32)",
  "status": "string"
}

Schema Definitions

Rate: object

price: integer (int32)

Price of the parking rate. Only populated if a rate is available for the submitted time span

status: string unavailable

Indicates parking rate is unavailable for submitted time span. Only populated when a parking rate is unavailable
